# plans/reusable/

Recurring or standing plans, rollouts, and process docs that are run repeatedly or read
across sessions (e.g. a periodic audit or a rollout runbook).

Not a terminal state: entries live here as long as they are reused, rather than moving to
`executed/` after a single run. Named `YYYYMMDD-HHMM-NN-<slug>.md` like other plans.
